# AWF Style Guide

Applies to all public-facing documents and code comments.

- No em-dashes. Use a colon, period or restructured sentence.
- No Oxford commas.
- Vendor-neutral language in all public-repo content.
- Market claims must be sourced and qualified.
- Status claims limited to what is provably true at time of writing.

Style compliance is a pre-merge check, not a test.
